无向图 \(G=<V, E>\)，其中 \(V \neq \varnothing\) 称为顶点集，其元素称为顶点，\(E\) 是 \(V \& V\) 的多重子集，称为边集，其元素称为无向边或边。有向图 \(D=<V, E>\)，其中 \(V\) 同无向图，\(E\) 是 \(V \times V\) 的多重子集，其元素称为有向边或边。有时用 \(G\) 泛指图（无向的或有向的），但 \(D\) 只表示有向图。用 \(V(G)(V(D))\)、\(E(G)(E(D))\) 分别表示 \(G(D)\) 的顶点集与边集。

“阶数”就是图里“点的个数”。

只有顶点没有边的图称为零图,只有一个顶点的零图称为平凡图。

环（Loop）：特指一条边，这条边的起点和终点是同一个顶点（自环）。

圈（Cycle）：特指一条路径，这条路径从某个顶点出发，经过至少三个不同的顶点后，又回到起点（闭合回路）。

对于无向图,若关联一对顶点的边多于一条,则称这些边为平行边。对于有向图,若关联一对顶点的方向相同的边多于一条,则称这些边为平行边。平行边的条数称作重数。既不含平行边,也不含环的图称为简单图。

图中任意两顶点至少有一条路径，那么这种图是连通图。两个顶点间必然有一条边，那么这种图是完全图。

有向图的连通性若略去有向图D中各边的方向所得无向图是连通图,则称D是弱通图(或连通图);若D中任何两个顶点至少一个可达另一个,则称D是单向连通图;若 D中任何两个顶点都是相互可达的,则称D是强连通图。强连通图一定是单向连通图,单向连通图一定是弱连通图。（弱连通而不单向连通的结构：V型\(a\to b \wedge a\to c\Rightarrow \,b\,\, not\to c\)）

连通且不含圈的简单图是树。平凡图又能叫平凡树。

正则图指的是每个顶点的度数都相同的无向简单图。

生成图：含所有顶点的子图；导出图：子集所含的点在原图中所关联的全部边都包含进去。

补图是指与原图具有相同顶点集，其边集由原图中 相对于原图所有顶点形成的完全图 不存在的边组成。

图的同构是指两个图在顶点和边的结构上完全相同，只是顶点标号可能不同。

通路（Path）是图中一系列连续相连的顶点序列，其中任意两个相邻顶点之间都有边相连。
回路（Cycle）是特殊的通路，其起点和终点相同。

一笔绘制一个路径，有起点，经过点，和终点。起终点会发出或接收一条路径，经过点则既发出一条路径又接收一条路径。所以一个笔画的起终点是有一个路径的，而经过点是有二个。
多个笔画叠加在一起，起点和终点如果在一起，就变成了经过点，1+1=2，它的通路是二，如果又作为其它笔画的经过点，就是二的倍数个通路，也就是偶数个通路。
如果不考虑笔画的方向，终点和终点，起点和起点，叠在一起的情况，和上文是一样的。
经过点和经过点叠加在一起，点的通路是偶数个。
起终点和经过点叠在一起，点的通路是奇数个。
过一点的通路有奇数个，这个点就是奇点。这意味着这个点是某个笔画的端点(起终点)

连通图如果能还原为笔画的叠加，那么由于笔画必有2个或0个端点，那么图的奇点必然是偶数个。

笔画的术语是迹。顶点的通路个数是度。

· 在标准图论定义中，一个单独的顶点被视为连通图（称为“平凡连通”）。 · 因为连通的定义是“任意两个顶点之间都有路径”，只有一个顶点时，条件“任意两个”为空真，所以成立。

一个图如果没有孤点，那么它一定能还原为迹。

如果有一条路径能经过图中所有通路有且仅有一次，那么这样的路径(最早由欧拉找到)就叫欧拉路径。如果欧拉路径是闭合的，就叫欧拉回路。 欧拉路径就是一个迹(笔画)能画出的路径。

如果有一条路径能经过图中所有顶点有且仅有一次，那么这样的路径(最早由哈密顿提出)就叫哈密顿路径。如果哈密顿路径是闭合的，就叫哈密顿回路。

在一条能走遍所有顶点的路线（哈密顿通路）上，每“跳过”一个顶点，最多只能让剩下的图断出一截。

连通分量

在无向图中，连通分量是图的一个极大连通子图。
· 连通：子图内任意两个顶点之间都存在路径。
· 极大：如果再加入原图中的任何一个额外顶点，子图就不再连通了。

割集就是“拆掉这组边（或点）之后，能把图一分为二的临界组合”。

点割集是图中删除后会增加连通分量的最小顶点集合。（如果从图中删除某个顶点集后，图变为不连通，且删除该集合的任意真子集仍保持连通，则该顶点集称为点割集）元素为割点。
边割集是图论中删除后会增加图连通分量的边的集合，且该集合是最小的使图不连通的边集。元素为桥。

矩阵

关联矩阵

如：

e1 e2 e3
v1 1 0 1
v2 1 1 0
v3 0 1 1

边的起点记作1，终点为-1，无向图中的端点都记作1。

可达矩阵

如三角形：

v1 v2 v3
v1 0 0 1
v2 1 0 0
v3 0 1 0

i到j的边记为1，对称的矩阵代表的是无向图。

邻接矩阵

v1 v2 v3
v1 0 0 1
v2 1 0 0
v3 0 1 0

\(v_i\) 到 \(v_j\) 的边数n为矩阵的元素 \(a_{ij}=n\)，对称的矩阵代表的是无向图。

把图的邻接矩阵 \(A\) 自乘上 \(l\) 次，得到的新矩阵 \(A^l\) 里，每个数字都精确地告诉你，从某个点出发，走 \(l\) 步能到达某个点的走法有多少种。

\(B_r\) 是这样定义的： \[ B_r = A + A^2 + A^3 + \cdots + A^r \]

也就是说，\(B_r\) 是前 \(r\) 个幂矩阵的“和”。

  • 既然 \(A^l\) 里每个元素代表“恰好走 \(l\) 步”的路径数。
  • 那么把它们加起来，自然就代表“走不超过 \(r\) 步”的路径总数。

迷宫

一个迷宫中的出入口，岔路口，拐角，死胡同的尽头视作顶点，顶点间的路就视作通路，那么迷宫就会变成一个连通图。

其中，死胡同的尽头，度数为1，拐角度数为2，岔路口的度数在2以上，令出入口的度数自动加1，因为通往迷宫外。

要找到一条迹连通出入口，一个做法是遍历所有的度数为1的点(死胡同的尽头)，抹去这些点和相关的通路，然后重复进行直到没有这样的点。 迭代的每次结果可以视作生成一个新的迷宫。

k核分解是将图中度数小于k的顶点及相关通路都迭代删去，直到没有。上文中寻找迷宫解法的方法就是2核分解。

通道标记的算法：

· 生成树：必须包含全部顶点。

若无向图G的生成子图T是一棵树,则称T为G的生成树。G在T中的边称为T的树枝,G不在T中的边称为T的弦。T的全体弦组成的集合的导出子图称为T的余树。注意,T的余树不一定是树,它可能不连通,也可能含回路。

基本回路与基本回路系统 设 T 是无向图 G 的生成树,对每条弦e,G 中有唯一一条 由e 和 T 的树枝构成的初级回路，称为对应弦e 的基本回路。G 中所有基本回路的集合称 为对应 T 的基本回路系统。每个弦对应一个回路。

基本割集与基本割集系统 对 T 的每条树枝a,G 中有唯一一个由 a 和 T 的弦构成的割集,称为对应树枝 a 的基本割集。G 中所有基本割集的集合称为对应 T 的基本割集系统。每个枝和唯一一组弦组成对应唯一的边割集

设无向图 \(G=<V, E>\)，\(|V|=n\)，\(|E|=m\)，则下面命题等价。

  1. \(G\) 连通且不含回路，即 \(G\) 是一棵树。

  2. \(G\) 的每对顶点之间有唯一的一条路径。

  3. \(G\) 连通且 \(m=n-1\)。

  4. \(G\) 中无回路且 \(m=n-1\)。

  5. \(G\) 中无回路，但在 \(G\) 中任何两个不相邻顶点之间加一条新边，所得图中含唯一的一条初级回路。

  6. \(G\) 连通且每条边都是桥。

任何一棵“正经的树”（无向树），只要它不止一个点，就至少有两个“末梢”（叶子）。

欧拉公式

设连通平面图有 V 个顶点，E 条边，F 个面（包括外部无限面）。
第一步：削去边，直到变成树（保持连通）
找到图中任意一个回路（环），删掉该回路上的一条边。
发生了什么变化？
· 顶点数 V：不变。
· 边数 E：减少 1。
· 面数 F：因为把两个相邻的面打通了，面数也减少 1。
结论：V - E + F 的值在删边前后保持不变（因为 E 和 F 同时减了 1，差值抵消了）。
第二步：直到没有回路（得到一棵生成树）
重复上述操作，直到图中不再有任何回路，此时图变成了一棵生成树。
· 对于一棵树，边数 E’ = V - 1。
· 对于一棵树，没有闭合区域，只有 1 个面（即外部无限面），所以 F’ = 1。
第三步：代入验证
此时：
V - E’ + F’ = V - (V - 1) + 1 = 1 + 1 = 2
因为在削边过程中，V - E + F 的值始终没有改变，所以原连通平面图也必然满足 V - E + F = 2。证毕！

最小生成树（最小成本树）

克鲁斯卡尔算法（Kruskal）——按边排序，避圈 这是考试中最爱考的方法（因为步骤清晰）： · 把所有的边按权值从小到大排序； · 从最小的边开始选，只要加入后不形成回路（不构成环），就保留；（构成环会浪费一条边，其中的最大者必须放弃。后来者定比先来者大。） · 直到选出 n-1 条边为止。

普里姆算法（Prim）——按点扩展，连最近 · 随便选一个顶点出发； · 每次从已选顶点集合出发，找到连接未选顶点的最短边，把那个新顶点拉进来； · 重复直到所有顶点都被包含

有向树及根树

若略去有向图所有边的方向所得无向图为无向树，则称 D 为有向树。 一棵非平凡的有向树 T，如果有一个顶点的入度为0，其余顶点的入度均为1，则称 T 为（有）根树。在根树中,入度为0的顶点称为树根。入度为1、出度为 0的顶点称为树叶。入度为 1、 出度不为0的顶点称为内点。树根与内点统称为分支点。在根树中,从树根到一个顶点的 通路长度称为该顶点的层数。顶点的最大层数称为树高。

一棵根树可以看作一个家族。非元祖的内点及其所有后代的导出子图可以成为根子树。

若对根树 T 中每层上的顶点指定顺序（排兄弟次序）,则称 T 为有序树。

根树的分类 设 \(T\) 为一棵根树。

  1. 若 \(T\) 的每个分支点至多有 \(r\) 个儿子，则称 \(T\) 为 \(r\) 叉树。若 \(T\) 的每个分支点都恰好有 \(r\) 个儿子，则称 \(T\) 为 \(r\) 叉正则树。此时又若 \(T\) 的所有树叶层数相同，则称 \(T\) 为 \(r\) 叉完全正则树。

  2. 有序的 \(r\) 叉树，称为 \(r\) 叉有序树。有序的 \(r\) 叉正则树称为 \(r\) 叉有序正则树。有序的 \(r\) 叉完全正则树称为 \(r\) 叉有序完全正则树。

Huffman 算法的核心思想是：每次把“最轻”的两块积木拼在一起，形成一块新的“重”积木，重复直到只剩一块。

具体步骤（手算模拟）：
列出所有叶子：假设有 5 个权值：5, 6, 2, 9, 7。
排序：2, 5, 6, 7, 9。
取最小两个合并：取 2 和 5，合并成 7（新节点权值为 2+5=7）。现在集合变为：6, 7（新）, 7（原）, 9。
再取最小两个合并：取 6 和 7（新），合并成 13。现在集合变为：7（原）, 9, 13。
再取最小两个合并：取 7 和 9，合并成 16。现在集合变为：13, 16。
最后合并：取 13 和 16，合并成 29（根节点）。

这棵树的形状：越早被合并的叶子（权值小的）在树的最底层；越晚合并的叶子（权值大的）在树的上层。这就实现了“高频在上，低频在下”。像一把锯子/梳子

一棵二叉树天然就等价于一套前缀码：

  • 把树的每一条左分支记为 0，每一条右分支记为 1。
  • 从根节点走到某个叶子节点，沿途经过的 0/1 序列，就是这个叶子对应的编码。

最佳前缀码 = 最优二叉树。

最短点遍历

Dijkstra（迪杰斯特拉）标号法，就是用来在“带权地图”上，快速找出从起点到所有其他点的最短路径（总权值最小）的一套标号打勾的步骤。它特别要求：所有路的长度（权值）都不能是负数。

  • 初始化：起点的距离标为 0（自己到自己），其他所有不通向的顶点标为无穷大（∞），通向的点标上权值。
  • 试探更新：从当前已经打勾的点出发，看它通向哪些邻居。如果走这条路能让邻居的距离变短，就更新（松弛） 邻居的临时标号。
  • 打勾确认：在所有还没有打勾的点里，找出临时标号最小的那个点。这个数字已经不可能再被其他路径缩短了（贪心思想），所以给它永久标号（打勾）。
  • 重复：以这个新打勾的点为跳板，继续向它的邻居试探……直到所有点都打完勾，或者到达了你想去的终点。

四色定理——任何平面地图（即平面图）的着色，最少只需要 4 种颜色。

二部图是一种图论模型，其顶点集可以划分为两个互不相交的子集，且每条边都连接两个不同子集的顶点。

二部图可以分成两个阵营（0和1）走出一个回路一定要进行偶数次阵营转变才回到原来阵营中的起点；反之如果有奇数长度的回路，从一点出发按步数的奇偶划分阵营，就会有同营相连，不是二部图。

图论中的匹配是指在一个图中，任意两条边都没有公共顶点的边的集合。匹配边的点是“饱和点”。匹配的类型包括最大匹配和完美匹配等。
最大匹配是指在一个图中，无法再通过添加其他边来增大匹配的情况。（具有最大边数的匹配）
完美匹配是指一个匹配中每个顶点都是匹配点，即所有顶点都恰好与一条匹配边相连。完美匹配一定是最大匹配，但并非所有图都存在完美匹配。

Hall定理：二部图中人少的一方任意分出一组，这组成员所匹配的对象总数不少于成员数，那么可以总体的完美匹配。（这样的条件称作相异性条件）

t条件 是相异性条件的极端版：一个阵营中所有顶点度数不小于t，而另一个阵营所有顶点度数不超过t，和Hall定理一样的结果。

寻找最大匹配——匈牙利算法：

  • 非匹配边相当于“新的配对提议”。
  • 匹配边相当于“旧的配对关系”。
    当我们沿着增广路径取反时，就像是一连串的“分手”和“牵手”：c抢了a的1，a就去抢b的2，b只好去抢空的3。最终，只有起点和终点（两个单身的人）改变了状态，路径中间的人只是换了个对象，但依然处于配对状态。这就导致了总配对数净增 1。
